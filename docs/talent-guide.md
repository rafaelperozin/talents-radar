# Guia do Talento — Como aparecer bem rankeado no Radar

## O que é o Radar?

O Radar é o sistema que as empresas usam para encontrar candidatos. Quando uma empresa busca alguém para uma vaga, ela descreve o perfil ideal e o Radar varre todos os talentos cadastrados, filtra quem atende os requisitos mínimos e **ordena os candidatos por compatibilidade**, de 0 a 100 pontos.

Quanto mais completo e preciso for o seu perfil, maior a chance de você aparecer no topo da lista — e na lista certa.

---

## O que acontece com o seu perfil

O Radar avalia seu cadastro em dois momentos:

### 1. Filtro eliminatório

Antes de qualquer pontuação, o sistema verifica se você atende os requisitos mínimos da vaga:

- **País / cidade** — você precisa estar no local buscado
- **Modalidade de trabalho** — sua preferência precisa coincidir com pelo menos uma aceita pela vaga
- **Habilidades técnicas mínimas** — quando a vaga exige um nível mínimo em uma skill, quem estiver abaixo é descartado
- **Soft skills obrigatórios** — quando listados como eliminatórios

Se você for descartado nessa etapa, **não aparece no resultado**, independente do quão bom seja o restante do seu perfil.

### 2. Ranqueamento por compatibilidade

Quem passa pelo filtro recebe uma nota de 0 a 100 baseada em:

- Nível real das suas habilidades técnicas vs. o esperado pela vaga
- Seniority
- Pretensão salarial
- Soft skills
- Palavras-chave no seu perfil (cargo, headline, descrição)

---

## Como preencher cada campo

### Localização

Informe o **país em inglês** (`Brazil`, `Portugal`, `United States`) e a cidade.

> O sistema é insensível a maiúsculas e acentos nas buscas — `"sao paulo"` encontra `"São Paulo"` — mas preencha corretamente assim mesmo para garantir consistência.

---

### Modalidade de trabalho

Escolha **a sua preferência real**. Os valores aceitos são:

| Valor    | Significado |
| -------- | ----------- |
| `remote` | 100% remoto |
| `hybrid` | Híbrido     |
| `onsite` | Presencial  |

> Dica: se você aceita tanto remoto quanto híbrido, prefira `remote` como primária — a maioria das buscas é por remoto e ampliar as opções aumenta suas chances.

---

### Habilidades técnicas

Esse é o campo com **maior peso** no ranqueamento. Preencha com cuidado.

#### Nível de cada skill

Seja honesto e preciso. O sistema compara o nível que você declarou com o nível esperado pela vaga:

| Nível          | Quando usar                                                           |
| -------------- | --------------------------------------------------------------------- |
| `beginner`     | Você conhece o básico e já usou na prática, mas ainda aprende muito   |
| `intermediate` | Você usa no dia a dia com autonomia e resolve a maioria dos problemas |
| `advanced`     | Você domina e resolve problemas complexos sem ajuda                   |
| `expert`       | Você é referência — ensina, define boas práticas, vai fundo           |

**Exagerar no nível não ajuda.** Se a vaga pede `intermediate` e você declarou `expert`, você pode ser rankeado acima de candidatos melhores — mas na entrevista a diferença vai aparecer. Preencha com honestidade.

**Subestimar também prejudica.** Se você é `advanced` mas declarou `intermediate`, você pode perder pontos desnecessariamente — ou pior, ser eliminado por não atingir o mínimo exigido.

#### Aliases e variantes

Muitas skills têm nomes alternativos. Quando você cadastrar uma skill, adicione as **variantes** pelos quais ela é conhecida:

| Skill        | Variantes recomendadas                      |
| ------------ | ------------------------------------------- |
| `TypeScript` | `ts`                                        |
| `Node.js`    | `nodejs`, `node`                            |
| `React`      | `reactjs`, `react.js`                       |
| `PostgreSQL` | `postgres`, `pg`                            |
| `AWS`        | `amazon web services`, `cloud aws`          |
| `Magento`    | `magento 2`, `adobe commerce`, `e-commerce` |

Quanto mais variantes você cadastrar, mais buscas diferentes vão te encontrar.

---

### Seniority

Escolha o nível que **melhor descreve onde você está hoje**, não onde quer chegar:

| Valor      | Cargo equivalente              |
| ---------- | ------------------------------ |
| `intern`   | Estagiário                     |
| `junior`   | Júnior                         |
| `mid`      | Pleno                          |
| `senior`   | Sênior                         |
| `staff`    | Staff / Principal              |
| `lead`     | Liderança Técnica / Lead       |
| `manager`  | Engineering Manager / Gerência |
| `director` | Diretor / VP                   |

**O sistema usa uma escala graduada** — você não precisa ser o nível exato da vaga para pontuar. Se a vaga quer `senior` e você é `staff` ou `mid`, ainda assim recebe uma pontuação parcial (75%). Dois níveis de distância valem 50%, três valem 25%.

---

### Pretensão salarial

O sistema usa **margens assimétricas** — estar abaixo do budget penaliza menos do que estar acima.

| Situação                              | Efeito no seu score                    |
| ------------------------------------- | -------------------------------------- |
| Sua pretensão = budget da vaga        | Pontuação máxima                       |
| Sua pretensão < budget da vaga        | Pontuação boa (mínimo 50%)             |
| Sua pretensão até 20% acima do budget | Pontuação decrescente                  |
| Sua pretensão > 20% acima do budget   | Zero pontos neste critério             |
| Sem pretensão informada               | 50% (neutro — não ajuda nem prejudica) |

> **Você não é eliminado por salário** — apenas perde pontos neste critério. Mas preencher a pretensão salarial coloca você em vantagem sobre quem deixou em branco, pois permite uma avaliação real de compatibilidade.

---

### Soft skills

Liste os soft skills que você realmente tem e que podem ser observados no trabalho. Exemplos comuns buscados por vagas:

`liderança` · `mentoria` · `comunicação` · `trabalho em equipe` · `organização` · `proatividade` · `autonomia` · `resolução de problemas` · `pensamento crítico` · `tomada de decisão` · `empatia` · `colaboração`

> Soft skills são buscados por correspondência exata de texto. Use termos simples e diretos — `"comunicação"` é melhor que `"excelente capacidade de comunicação interpessoal"`.

---

### Headline e cargo

Seu cargo (`role`) e headline são incluídos na busca por **palavras-chave** das vagas. Isso significa que termos como `fintech`, `e-commerce`, `B2B`, `microsserviços`, `open source`, `distributed systems` no seu headline podem fazer você aparecer em buscas específicas.

**Exemplos de headline com boas palavras-chave:**

- `"React/TS specialist — design systems, performance, a11y"`
- `"NestJS, Postgres, distributed systems — backend pleno"`
- `"Fullstack Engineer — React, Node, AWS, Terraform, squad lead"`

> Seja descritivo sem exagerar. O sistema busca por presença de termos — não avalia qualidade do texto.

---

## Checklist do perfil ideal

Antes de finalizar seu cadastro, verifique:

- [ ] País e cidade preenchidos corretamente (país em inglês)
- [ ] Modalidade de trabalho definida com sua preferência real
- [ ] Todas as suas principais skills cadastradas com nível honesto
- [ ] Aliases e variantes adicionados às skills com nomes alternativos comuns
- [ ] Seniority refletindo sua posição atual
- [ ] Pretensão salarial informada
- [ ] Soft skills relevantes listados (5–10 é um bom número)
- [ ] Headline descritivo com contexto de experiência e tecnologias

---

## Perguntas frequentes

**Preciso listar todas as minhas skills?**
Não é obrigatório, mas quanto mais completo, maior a chance de aparecer em buscas relevantes. Priorize as skills que você usaria num próximo trabalho.

**Devo colocar skills que aprendi há anos mas não uso mais?**
Só se ainda se sente confortável em trabalhar com elas. O sistema pode te ranquear alto para vagas que incluem essas skills como requisito, e você vai precisar correspondê-las na entrevista.

**Minha seniority está entre dois níveis — qual escolho?**
Escolha o mais baixo dos dois. É melhor surpreender positivamente do que criar expectativas que você ainda está desenvolvendo.

**Por que não apareço em nenhuma vaga?**
Provavelmente você está sendo eliminado pelo filtro obrigatório. Verifique se seu país/cidade, modalidade de trabalho e habilidades técnicas estão preenchidos corretamente e com os valores aceitos pelo sistema.

**Posso ter mais de uma pretensão dependendo da vaga?**
O cadastro tem um único valor de pretensão. Coloque o valor que representa sua expectativa para a maioria das oportunidades que você buscaria.
